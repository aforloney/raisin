# Raisin v0.1 Scope >>  v0.1-scope.md

## Purpose
Reduce implementation rework caused by hidden assumptions.

## User
Software engineers designing technical changes.

## Workflow
Decision → Assumptions → Evidence → Decision → Plan

## Readiness
Proceed when uncertainty is visible and accepted.

## Success Metric
Reduced significant re-planning after implementation begins.

## Explicitly Not Included
- REOS
- adapters
- compliance
- schemas
- autonomous execution
