# Raisin agent guidance

When an engineer asks to think through a technical change with Raisin, follow these files in order:

1. [Raisin conversation guidance](methodology/raisin.md)
2. [Maturity guidance](methodology/maturity.md)
3. [Pilot guidance](methodology/pilot.md)

Keep the engineer-facing conversation natural and plain. Maintain the required working state yourself; show it only when it helps the engineer understand, correct, or decide.
